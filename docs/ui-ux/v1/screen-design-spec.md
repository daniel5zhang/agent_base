# 核心屏幕设计规格

版本：v1  
日期：2026-07-01

## 1. App Shell

### 1.1 布局

桌面端布局：

```text
┌───────────────┬────────────────────────────┬───────────────────┐
│ 左侧会话栏     │ 中间 Agent Thread           │ 右侧业务面板        │
│ 240-280px     │ fluid / max readable width  │ 360-640px          │
└───────────────┴────────────────────────────┴───────────────────┘
```

规则：

- 左侧默认展开，支持折叠。
- 右侧默认关闭，只有点击 Artifact 卡片或用户手动打开后显示。
- 中间主对话区始终优先保持阅读宽度。
- 右侧打开后不应把主对话压缩到不可读状态。
- 分隔线和 resize handle 使用低对比样式，不能形成突兀进度条感。

### 1.2 主题

必须支持：

- Light
- Dark
- 跟随系统

实现要求：

- 优先沿用 shadcn/ui theme token。
- 不在组件内写死大面积色值。
- 业务 Artifact Renderer 也必须使用统一 token。

## 2. 左侧会话栏

### 2.1 展开态

结构：

```text
品牌区
新建任务
今天
  会话 A
  会话 B
昨天
  会话 C
更早
  会话 D
底部：设置与模型
```

交互：

- 点击会话只切换当前 Thread，不应触发会话列表整体刷新。
- 当前会话显示选中背景。
- 会话 hover 显示更多按钮。
- 更多按钮支持重命名、删除，后续可支持归档。
- 新建任务创建空 Thread 并进入空态。

### 2.2 折叠态

结构：

```text
品牌图标
新建任务图标
设置图标
```

交互：

- 点击品牌图标展开左侧栏。
- hover 显示 tooltip。
- 折叠动画与右侧面板折叠动画保持一致。

## 3. 中间 Agent Thread

### 3.1 空态

目标形态：

```text
今天要处理什么？

[ Composer ]
```

规则：

- 不展示大段说明。
- 不展示假任务或假结果。
- Composer 位置居中偏下，保持 assistant-ui Base 风格。

### 3.2 消息结构

用户消息：

```text
                         [用户输入 bubble]
```

助手消息：

```text
[执行过程 v]
助手回复正文
[结果卡片 / 审批卡片]
[复制] [重试] [...]
```

规则：

- 执行过程默认折叠。
- 执行过程字体颜色低于正文，不抢主内容。
- 执行过程必须绑定到本轮 assistant 消息。
- 模型输出采用流式打字机效果。
- 工具调用状态可动态更新，但完成后默认折叠。

### 3.3 Artifact 结果卡片

卡片内容：

```text
标题：2026 年惠民保项目总保费
摘要：总保费 12.36 亿元
来源：问数插件 / 数仓
状态：已完成
操作：打开结果
审计：evt_xxx
```

交互：

- 点击卡片打开右侧业务面板。
- 如果对应 Tab 已打开，则激活。
- 如果 Tab 已关闭，则重新打开。
- 卡片留在主对话中，作为历史结果入口。

### 3.4 审批摘要卡片

卡片内容：

```text
标题：需要申请数据权限
范围：全国惠民保保费，2026 年
原因：超出当前数据范围
状态：待提交 / 审批中 / 已通过 / 已拒绝
操作：查看详情 / 提交申请 / 继续执行 / 重新提交
```

规则：

- 审批详情不在右侧面板展示。
- 点击详情进入设置页审批详情。
- 审批通过后不自动继续，必须用户点击“继续执行”。

## 4. Composer

基础规格：

- Enter 发送。
- Option + Enter 换行。
- `+` 添加插件能力。
- 附件按钮添加文件。
- 发送中显示停止按钮。
- 禁用状态显示清晰原因。

`+` 菜单：

```text
已授权插件
  问数
  理赔
  投保
本地能力
  文件读取
  本地数据分析
申请更多插件权限
```

规则：

- 未授权内部业务插件不在快捷列表显示。
- 自然语言触发未授权能力时，在主对话展示授权申请卡片。
- 插件能力选择只影响本轮 Run，不改变全局设置。

## 5. 右侧业务 Artifact 面板

### 5.1 面板结构

目标结构：

```text
┌──────────────────────────────┐
│ [Tab A] [Tab B]        [+]   │
├──────────────────────────────┤
│ Artifact Renderer Host        │
│                              │
└──────────────────────────────┘
```

规则：

- 顶部不展示“业务面板”大标题。
- Tab 行是主要导航。
- `+` 用于手动添加或从可用 Artifact 中选择打开，不创建假面板。
- 面板内容区只渲染 Artifact。

### 5.2 Tab 行为

- Tab hover 右侧显示关闭 `x`。
- 关闭当前 Tab 后切换到相邻 Tab。
- 关闭最后一个 Tab 后显示空态。
- 关闭不删除后端 Artifact。
- 点击主对话卡片可恢复已关闭 Tab。

### 5.3 空态

空态文案：

```text
从对话中的结果卡片打开业务内容
```

规则：

- 不展示 mock 数据。
- 不展示“运行证据 / Artifact / 事件 / 诊断”等旧占位内容。

### 5.4 Renderer

首批 renderer：

| renderer_hint | 用途 |
| --- | --- |
| `table` | 明细表格 |
| `metric_summary` | 指标摘要 |
| `chart` | 图表 |
| `report` | 报告预览 |
| `form` | 受控业务表单 |
| `json_fallback` | 未识别结构 fallback |

安全规则：

- 不执行模型生成 JS。
- 不渲染任意 HTML。
- 只渲染受控 schema。
- 下载、导出、复制操作必须权限判断和审计。

## 6. 设置页面

### 6.1 页面结构

设置页是完整页面，不是弹窗。

```text
设置与模型
├── 概览
├── 模型
├── 插件
├── 审批
├── 账号与权限
├── 审计
└── 诊断
```

### 6.2 模型页

内容：

- 当前默认模型。
- Provider 列表。
- 新增 Provider。
- 编辑 Provider。
- API Key 脱敏展示。
- 有权限时短时明文查看。
- 测试连接。
- 刷新模型列表。
- 用量统计入口。

### 6.3 插件页

内容：

- 按业务类型分组：问数、理赔、投保、查询、办公等。
- 状态筛选：可用、未授权、已启用、已停用、需要审批。
- 插件详情。
- 申请授权。
- 启用 / 停用 / 使用。
- 管理员管理。
- 插件审计。

### 6.4 审批页

内容：

- 我的申请。
- 待我审批。
- 已完成。
- 审批详情。
- 通过 / 拒绝 / 重新提交 / 继续执行。

### 6.5 审计页

内容：

- 按时间、用户、会话、Run、插件、操作类型筛选。
- 审计详情。
- 脱敏展示。
- 有权限可查看明文。
- 导出审计记录。
